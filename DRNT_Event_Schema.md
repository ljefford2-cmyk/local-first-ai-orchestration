# DRNT Canonical Event Schema

**Version 2.0** · March 2026 · `schema_version: "2.0"` · Build Target: Docker + JSONL on WSL2/Ubuntu

Consolidates and supersedes Spec 1 v1.5 event definitions. This is the single reference for every event the DRNT governance layer can emit. A developer implementing logging writes to this schema and nothing else.

---

## 1. Storage

**Format:** JSONL. One JSON object per line. One file per UTC calendar day.

```
/var/drnt/audit/drnt-audit-2026-03-26.jsonl
```

Bind-mounted from host filesystem. Only the **log writer** container has write access. All other containers mount read-only. The orchestrator never writes directly to audit files — it sends events to the log writer over a Unix domain socket.

**Integrity chain:** Every event carries `prev_hash` — the SHA-256 hex digest of the previous event's complete JSON line (UTF-8, no trailing newline). The genesis seed is SHA-256 of the literal string `"DRNT-GENESIS"`:

```
9cd9da92046a4de28d7cec92f9559e542ba6a52af9fe44513bd26b0c2838a4d4
```

New daily files chain from the last event of the previous day. If no previous day exists, genesis seed is used.

**Midnight rollover:** Log writer determines target file by UTC date of `committed_at`. Finish current event to current file, record hash of last line, open new file chaining from that hash. Log writer is single-threaded with respect to writes.

**Retention (v1):** All files kept indefinitely. Querying via `jq`, `grep`, `tail -f`.

## 2. Event Envelope

Every event carries these fields. No exceptions. No optional envelope fields.

| Field | Type | Description |
|-------|------|-------------|
| `schema_version` | `string` | `"2.0"`. Consumers must check before parsing. |
| `event_id` | `UUIDv7` | Assigned by log writer at commit time. Identity of the log record. |
| `source_event_id` | `UUIDv7` | Generated by emitter at occurrence time. Identity of the logical event. Stable across retransmissions. Used for deduplication. |
| `timestamp` | `ISO 8601 µs` | UTC. When the event occurred. Set by emitter. |
| `committed_at` | `ISO 8601 µs` | UTC. When the log writer durably wrote it. The gap is delivery latency. |
| `event_type` | `string enum` | Dot-namespaced. Determines payload structure. |
| `job_id` | `UUIDv7 │ null` | The job this event belongs to. Null for system-level events. |
| `parent_job_id` | `UUIDv7 │ null` | Points to original job when spawned by redirect or escalate override. Null for user-initiated jobs. |
| `capability_id` | `string │ null` | Governing capability for this job. Set at `job.classified`, carried through all subsequent events, never mutated. Null for system events and `job.submitted` (pre-classification). |
| `wal_level` | `int (-1..3) │ null` | Effective WAL of governing capability at event time. `-1` = suspended. Null when `capability_id` is null. |
| `source` | `string enum` | `orchestrator │ context_packager │ egress_gateway │ watch_app │ phone_app │ human │ system │ log_writer │ worker_egress_proxy │ sandbox_runtime` |
| `prev_hash` | `hex64` | SHA-256 of previous event's complete JSON line. |
| `durability` | `string enum` | `"durable"` or `"best_effort"`. Set by emitter per event type. Durable events require fsync + ACK before the gated action proceeds. |
| `payload` | `object` | Event-type-specific data. Fixed per `event_type`. All fields present in every instance; null for inapplicable values, never omitted. |

### 2.1 Capability Scoping

One job, one governing capability. Classification establishes it; it never changes within a job. Auxiliary capabilities emit their own `wal.permission_check` events with their own `capability_id` in the **payload** — the **envelope** always reflects the governing capability.

Failures attribute to the capability that failed (`failing_capability_id`), not the governing capability. Three sentinel values are excluded from WAL failure counters: `"egress_config"` (route/policy misconfiguration), `"egress_connectivity"` (transport failure), and `"worker_sandbox"` (sandbox violation, Spec 6). Only registered `capability_id` values increment failure counts.

### 2.2 Example Envelope

```json
{
  "schema_version": "2.0",
  "event_id": "019508a3-7b2c-7def-8a12-4b3c5d6e7f89",
  "source_event_id": "019508a3-7b2c-7def-8a12-4b3c5d6e7f80",
  "timestamp": "2026-03-26T14:32:07.482193Z",
  "committed_at": "2026-03-26T14:32:07.491042Z",
  "event_type": "job.dispatched",
  "job_id": "019508a3-7b2c-7def-8a12-4b3c5d6e7f88",
  "parent_job_id": null,
  "capability_id": "route.cloud.claude",
  "wal_level": 0,
  "source": "egress_gateway",
  "prev_hash": "a3f2c8...9c7d",
  "durability": "durable",
  "payload": { }
}
```

## 3. Event Catalog

Events are grouped by domain. **D** = durable (fsync + ACK required). **BE** = best-effort.

### 3.1 Job Lifecycle

Every job produces at minimum `job.submitted` and one terminal event: `job.delivered`, `job.failed`, or `job.revoked`.

---

#### `job.submitted`

**When it fires:** User submits a request from watch or phone.

**Trigger:** Voice command, text input, or tap action on a device.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `raw_input` | `string` | The user's request as received. |
| `input_modality` | `enum` | `voice │ text │ tap` |
| `device` | `enum` | `watch │ phone` |
| `request_category` | `string │ null` | Null at submission (pre-classification). |

**Example payload:**
```json
{
  "raw_input": "Compare Claude and GPT-4 on summarizing legal briefs",
  "input_modality": "voice",
  "device": "watch",
  "request_category": null
}
```

---

#### `job.classified`

**When it fires:** Local model finishes classifying the request and assigns a governing capability.

**Trigger:** Orchestrator completes routing classification.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `request_category` | `string` | Classified task type. |
| `confidence` | `float 0–1` | Classification confidence. |
| `local_capable` | `bool` | Whether the local model can handle this alone. |
| `routing_recommendation` | `enum` | `local │ cloud │ multi` |
| `candidate_models` | `string[]` | Models considered for dispatch. |
| `governing_capability_id` | `string` | The capability that will own this job. |
| `declared_pipeline` | `array` | Snapshot of the governing capability's pipeline: `[{capability_id, dependency_mode}]`. Dependency modes: `required │ optional │ best_effort`. |

**Example payload:**
```json
{
  "request_category": "comparative_analysis",
  "confidence": 0.92,
  "local_capable": false,
  "routing_recommendation": "multi",
  "candidate_models": ["claude-sonnet-4-20250514", "gpt-4o"],
  "governing_capability_id": "route.multi",
  "declared_pipeline": [
    {"capability_id": "context.package", "dependency_mode": "required"},
    {"capability_id": "memory.read", "dependency_mode": "optional"},
    {"capability_id": "notify.phone", "dependency_mode": "best_effort"}
  ]
}
```

---

#### `job.queued`

**When it fires:** Job cannot be dispatched immediately.

**Trigger:** Connectivity loss, rate limit hit, or capacity constraint.

**Durability:** BE

| Field | Type | Description |
|-------|------|-------------|
| `reason` | `enum` | `connectivity │ rate_limit │ capacity` |
| `position` | `int` | Queue position. |
| `estimated_wait_ms` | `int │ null` | Estimated wait time. Null if unknown. |

---

#### `job.dispatched`

**When it fires:** Egress gateway confirms outbound transmission, **before** the model responds.

**Trigger:** Gateway completes all validation checks (Section 3.7) and sends the request.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `target_model` | `string` | Model receiving the request. |
| `route_id` | `string` | Egress registry reference. Never contains secrets. |
| `context_package_id` | `string` | Cross-ref to `context.packaged` event. |
| `wal_permission_check_ref` | `string` | `source_event_id` of the authorizing `wal.permission_check`. |
| `retry_sequence` | `int` | 0-indexed retry counter. |
| `same_capability_retry` | `bool` | V1: always `true`. Cross-capability retry is v2. |
| `assembled_payload_hash` | `string` | SHA-256. Must match `context.packaged` hash. |
| `egress_config_hash` | `string` | SHA-256 of `egress.json` as loaded by gateway. |

**Example payload:**
```json
{
  "target_model": "claude-sonnet-4-20250514",
  "route_id": "claude-sonnet-default",
  "context_package_id": "019508a3-8c1d-7abc-9012-3d4e5f6a7b8c",
  "wal_permission_check_ref": "019508a3-7b2c-7def-8a12-4b3c5d6e7f81",
  "retry_sequence": 0,
  "same_capability_retry": true,
  "assembled_payload_hash": "b4c7e2...f1a3",
  "egress_config_hash": "d8f1a9...c2b4"
}
```

**Note:** For `route.multi`, one `job.dispatched` per model (same `job_id`, different `route_id`, same `assembled_payload_hash`).

**Irreversibility boundary:** `job.dispatched` is the point where data has left the local environment. Before this event, overrides prevent egress. After this event, overrides control response handling but cannot recall data already transmitted. This boundary is the same in all three governance artifacts.

---

#### `job.response_received`

**When it fires:** Orchestrator acknowledges that sufficient response data exists to advance the job pipeline. In single-model workflows, this fires after the corresponding `model.response`. In multi-model workflows, this fires once the orchestrator determines which response to proceed with.

**Trigger:** Orchestrator's pipeline logic confirms a usable response is available.

**Durability:** D

**Relationship to `model.response`:** `model.response` is the canonical per-model event — one per model per dispatch, authoritative for latency, billing, and result-store linkage. `job.response_received` is the orchestration-layer acknowledgment that the job can proceed to delivery. In single-model flows, the two are sequential and carry the same `result_id`. In multi-model flows, `model.response` fires once per model; `job.response_received` fires once for the selected response.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `model` | `string` | Responding model. |
| `latency_ms` | `int` | Round-trip time. |
| `token_count_in` | `int` | Input tokens billed. |
| `token_count_out` | `int` | Output tokens billed. |
| `cost_estimate_usd` | `float │ null` | Estimated cost. Null if unknown. |
| `truncated` | `bool` | Whether the response was truncated. |
| `result_id` | `UUIDv7` | Pointer to results store. |
| `response_hash` | `string` | SHA-256 of exact bytes stored for `result_id`. |

---

#### `job.delivered`

**When it fires:** Result is presented to the user on their device.

**Trigger:** Device app confirms display of result.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `delivery_target` | `enum` | `watch │ phone` |
| `display_format` | `string` | How the result was rendered. |
| `acknowledged` | `bool` | Whether the user acknowledged receipt. |
| `review_gate_applied` | `enum` | `none │ pre_delivery │ post_action` |

---

#### `job.failed`

**When it fires:** Job execution did not complete, or pre-delivery cancel/redirect terminated it.

**Trigger:** Error, timeout, policy block, or override-induced termination.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `error_class` | `enum` | `timeout │ rate_limit │ model_error │ context_error │ policy_block │ override_cancel │ override_redirect │ internal` |
| `error_message` | `string` | Human-readable description. |
| `retryable` | `bool` | Whether the orchestrator considers this retriable. |
| `retry_count` | `int` | Number of retries already attempted. |
| `failing_capability_id` | `string` | The capability whose action failed. May differ from envelope. Sentinels `"egress_config"` / `"egress_connectivity"` / `"worker_sandbox"` for infrastructure failures (excluded from WAL counters). |

**`error_class` semantics:**
- `policy_block` — a `wal.permission_check` returned `blocked` or an egress validation check failed. The job never reached a model.
- `override_cancel` — the human cancelled the job before delivery. The original job terminates; no successor is spawned.
- `override_redirect` — the human redirected the job. The original job terminates; a successor job is spawned with a new governing capability (see `job_id`/`parent_job_id` linkage).

---

#### `job.revoked`

**When it fires:** A previously delivered job is withdrawn by the human.

**Trigger:** Post-delivery cancel, escalation supersede, or offline retroactive withdrawal.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `reason` | `enum` | `user_cancel │ escalation_supersede │ offline_retroactive` |
| `override_source_event_id` | `string` | `source_event_id` of the triggering `human.override`. |
| `successor_job_id` | `string │ null` | Replacement job if redirect/escalate. Null for pure cancellation. |
| `memory_objects_superseded` | `string[]` | IDs of memory objects marked superseded. |

**Design note:** Revocation does not delete prior events. `job.delivered` and `model.response` remain in the log. This is append-only — the revocation records the human's post-hoc decision.

### 3.2 Context Packager Events

Every outbound cloud dispatch produces exactly one `context.packaged` event. `context.strip_detail` fires once per non-pass transformation.

---

#### `context.packaged`

**When it fires:** Context Packager finishes assembling and sanitizing the outbound payload.

**Trigger:** Packager completes sensitivity scan and strip operations for a dispatch.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `context_package_id` | `UUIDv7` | Unique ID for this package. Cross-referenced by `job.dispatched`. |
| `job_id` | `string` | The job this package serves. |
| `governing_capability_id` | `string` | Governing capability requesting the package. |
| `governing_wal_level` | `int` | WAL of governing capability. |
| `context_package_wal_level` | `int` | WAL of `context.package` capability. |
| `effective_packaging_level` | `int` | `min(governing WAL, context.package WAL)`. Most restrictive wins. |
| `fields_included` | `string[]` | `field_id` values that passed through. |
| `fields_stripped` | `string[]` | `field_id` values that were stripped or generalized. |
| `fields_blocked` | `string[]` | `field_id` values blocked as unclassifiable (default-deny). |
| `memory_objects_attached` | `int` | Count of memory objects included. |
| `memory_retrieval_ms` | `int` | Time spent on vector memory retrieval. |
| `outbound_token_estimate` | `int` | Estimated token count of assembled payload. |
| `assembled_payload_hash` | `string` | SHA-256 of the assembled payload. Verified by egress gateway. |
| `context_object_hash` | `string` | SHA-256 of the full context object (stored separately in `/var/drnt/context/`). |
| `exclusion_summary` | `object` | Pre-scan gate counts: `{local_only: int, below_confidence: int, over_budget: int}`. |

---

#### `context.strip_detail`

**When it fires:** One event per field that was stripped, generalized, or blocked by the sensitivity scanner.

**Trigger:** Sensitivity scanner identifies a field requiring transformation.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `context_package_id` | `string` | Package this strip belongs to. |
| `field_id` | `UUIDv7` | The specific field affected. |
| `field_name` | `string` | Human-readable field name. |
| `span` | `object │ null` | `{start, end}` for sub-field precision. Null if entire field. |
| `original_class` | `enum` | `name │ location │ date │ financial │ credential │ custom │ unclassifiable` |
| `classification_status` | `enum` | `matched │ safe_none │ unclassifiable` |
| `action` | `enum` | `stripped │ generalized │ blocked` |
| `reason` | `string` | Why this action was taken. |
| `detected_by` | `enum` | `context_packager │ egress_gateway` |
| `failing_capability_id` | `string │ null` | Set to `"context.package"` only on egress gateway catches (where the Packager should have caught it but didn't). Null on Packager's own catches (conservative success). Triggers strip-failure trust consequences when non-null. |

### 3.3 WAL State Events

WAL state is per-capability. These events record state machine transitions and permission enforcement.

---

#### `wal.promoted`

**When it fires:** A capability's WAL level increases.

**Trigger:** Human edits `desired_wal_level` in `capabilities.json` and restarts. Startup reconciliation detects desired > effective.

**Durability:** D (fsync + ACK before effective level changes)

| Field | Type | Description |
|-------|------|-------------|
| `capability_id` | `string` | The promoted capability. |
| `capability_name` | `string` | Human-readable name. |
| `from_level` | `int (-1..3)` | Previous effective WAL. `-1` for suspended recovery. |
| `to_level` | `int (0..3)` | New effective WAL. |
| `criteria_met` | `object` | Snapshot of evidence that justified promotion. |
| `approved_by` | `string` | Always `"human"` in v1. |

**Example payload:**
```json
{
  "capability_id": "route.cloud.claude",
  "capability_name": "Claude Cloud Dispatch",
  "from_level": 0,
  "to_level": 1,
  "criteria_met": {
    "evaluable_outcomes": 34,
    "calendar_span_days": 12,
    "approval_score": 0.97,
    "strip_failures": 0,
    "cancel_redirect_overrides": 0
  },
  "approved_by": "human"
}
```

---

#### `wal.demoted`

**When it fires:** A capability's WAL level decreases (including suspension to `-1`).

**Trigger:** Automatic (3 failures in 24h, strip failure, model change, override) or manual (human edit + restart).

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `capability_id` | `string` | The demoted capability. |
| `capability_name` | `string` | Human-readable name. |
| `from_level` | `int (-1..3)` | Previous effective WAL. |
| `to_level` | `int (-1..3)` | New effective WAL. `-1` = suspended. |
| `trigger` | `enum` | `error │ anomaly │ override │ model_change │ manual │ config_reconcile │ strip_failure` |
| `incident_ref` | `string │ null` | `source_event_id` of the triggering incident. Null for manual/config_reconcile. |

**Demotion rules:**
- Strip failure (egress catch) → **suspend** `context.package` (to `-1`)
- Cancel/redirect override on WAL-1+ → **demote 1** (conditional — no demotion if prior sentinel egress failure)
- 3 failures in rolling 24h → **demote 1** (attributed to `failing_capability_id`)
- Model change → **WAL-0** for all affected capabilities
- On any demotion: counters cleared, `first_job_date` nulled, `last_reset` set

---

#### `wal.permission_check`

**When it fires:** Every time the system evaluates whether a capability may perform a requested action.

**Trigger:** Orchestrator processes a pipeline step or governing action.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `capability_id` | `string` | The capability being checked (may differ from envelope for auxiliary checks). |
| `job_id` | `string` | The job requesting the action. |
| `requested_action` | `string` | Canonical action from the action enum (e.g., `dispatch_cloud`, `package_context`, `write_memory`). |
| `current_level` | `int` | Capability's current effective WAL. |
| `required_level` | `int` | Minimum WAL needed for this action with this gate. |
| `result` | `enum` | `allowed │ blocked │ held` |
| `delivery_hold` | `bool` | True when result is `allowed` but `deliver_result` must apply `pre_delivery` gate. |
| `hold_type` | `enum │ null` | `pre_action │ cost_approval │ on_accept │ null`. Present only when result is `held`. |
| `block_reason` | `string │ null` | Explanation when blocked. |
| `dependency_mode` | `enum │ null` | `required │ optional │ best_effort │ null`. Null for governing capability checks. Determines block/skip/continue on failure. |

**Three-state result semantics:**
- `allowed` — action proceeds (may still have `delivery_hold: true`)
- `blocked` — capability's WAL does not permit this action; architectural denial
- `held` — action blocked pending human input; `hold_type` specifies what is needed

**`delivery_hold` clarification:** `delivery_hold: true` means the requested action (e.g., `dispatch_cloud`) is allowed to execute and incurs real cost, but downstream user-visible delivery remains blocked pending human review. This is the canonical encoding of `pre_delivery` — the gate holds the *result*, not the *action*. `pre_delivery` is never encoded as a `held` result.

### 3.5 Human Decision Events

---

#### `human.reviewed`

**When it fires:** User makes a decision about a job result — either releasing a pre-delivery hold (WAL-0/WAL-1) or providing post-delivery feedback (WAL-2+).

**Trigger:** User accepts, modifies, rejects, or resubmits from watch or phone. At WAL-0/WAL-1, this event releases a delivery hold and authorizes `job.delivered`. At WAL-2+, this event records post-delivery feedback on an already-delivered result.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `job_id` | `string` | The job under review. |
| `decision` | `enum` | `accepted │ modified │ rejected │ resubmitted │ approved_hold` |
| `review_gate_context` | `enum` | `pre_delivery_hold │ post_delivery │ pre_action_hold` — distinguishes hold release from post-delivery feedback. |
| `device` | `enum` | `watch │ phone` |
| `review_latency_ms` | `int` | Time from result availability to decision. |
| `modification_summary` | `string │ null` | What was changed (modified only). |
| `derived_from` | `string │ null` | `result_id` of original result (modified only). |
| `modified_result_id` | `string │ null` | `result_id` of the human-edited artifact (modified only). |
| `modified_result_hash` | `string │ null` | SHA-256 of modified artifact (modified only). |

**`decision: approved_hold`** is used when the human approves a `pre_action` hold (e.g., routing choice at WAL-0). It releases the held action but does not constitute feedback on result quality. It is not counted in promotion scoring.

**Scoring for WAL promotion:** accepted = 1.0, modified = 0.5, rejected = 0.0, resubmitted = 0.0, approved_hold = not scored, auto_delivered (WAL-2+ not overridden within 24h) = 1.0.

---

#### `human.override`

**When it fires:** User intervenes to change the system's routing, execution, or delivery.

**Trigger:** Cancel, redirect, modify, or escalate action from watch or phone.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `job_id` | `string` | The job being overridden. |
| `override_type` | `enum` | `cancel │ redirect │ modify │ escalate` |
| `target` | `string` | What was overridden (e.g., `routing`, `response`, `wal_action`). |
| `detail` | `string` | Specifics (e.g., `route.multi` for escalate-to-multi-model). |
| `device` | `enum` | `watch │ phone` |

**Override consequences:**
- `cancel` on viable route → demote governing capability by 1
- `redirect` on viable route → demote governing capability by 1
- `cancel`/`redirect` after sentinel egress failure → no demotion
- `modify` → no demotion (quality feedback; score 0.5)
- `escalate` → no demotion (quality feedback; score 0.0)
- `redirect`/`escalate` → terminate original job, spawn successor with new `job_id` and `parent_job_id`

---

#### `human.feedback`

**When it fires:** User provides structured feedback on a job result.

**Trigger:** User submits feedback at any of three levels.

**Durability:** D

| Field | Type | Description |
|-------|------|-------------|
| `job_id` | `string` | The job receiving feedback. |
| `level` | `int (1–3)` | Feedback granularity level. |
| `level_1_signal` | `enum` | `used │ resubmitted │ dismissed │ delayed` |
| `level_2_category` | `enum │ null` | `wrong_model │ missing_context │ wrong_format │ too_verbose │ too_brief │ other` |
| `level_3_voice_text` | `string │ null` | Raw voice transcription. Stored as-is. |

### 3.8 Worker Execution Events (Spec 6)

These events govern the execution silo — the runtime environment where worker agents operate. They follow the same envelope as all other events. All worker events are durable. Source enum values: `worker_egress_proxy` (egress events) and `sandbox_runtime` (violation events).

Worker events carry the governing capability's `capability_id` and `wal_level` in the envelope, consistent with Section 2.1 (one job, one governing capability). `agent_type_id` and `agent_instance_id` are payload fields, not envelope fields.

#### `worker.sandbox_violation`

**When it fires:** Sandbox runtime catches a prohibited action at the OS/container level.

**Trigger:** Landlock, seccomp, or network namespace enforcement blocks an agent action.

**Durability:** D

|-------|------|-------------|
| `agent_type_id` | `string` | Agent type that caused the violation. |

**Note:** `job_id` in the envelope may be null if the violation occurs while the worker is idle between dispatches. When null, this is a system-level integrity event. When non-null, the event links to the job's audit trail but the sentinel ensures no WAL demotion fires.

## 3.9 Replay and Ordering

**Integrity chain order** is by `committed_at` — the sequence in which the log writer durably wrote events. This is the storage truth.

**Semantic occurrence order** is inferred from `timestamp` — the time the emitter reports the event happened. For co-located components, `timestamp` and `committed_at` are nearly identical. For watch/phone events queued during offline periods, the gap may be significant.

**Conflict resolution:** Storage replay always follows hash-chain order (`committed_at`). Causal analysis uses `timestamp` when the source is trusted (i.e., the emitter's clock is reasonable). Late-arriving device events that logically precede already-committed orchestration events are stored in chain order but their `timestamp` enables correct causal reconstruction.

## 4. Event Sequences

### 4.1 Minimal Job (WAL-0, Single Cloud Model)

```
1.  job.submitted         (phone_app)
2.  job.classified        (orchestrator) → sets governing_capability_id
3.  wal.permission_check  (context.package — auxiliary, dependency_mode: required)
4.  context.packaged      (context_packager)
5.  wal.permission_check  (route.cloud.claude — governing, select_model → held, pre_action)
6.  human.reviewed        (human) → approved_hold (approves routing choice)
7.  wal.permission_check  (route.cloud.claude — governing, dispatch_cloud → allowed, delivery_hold: true)
8.  job.dispatched        (egress_gateway) → data leaves local
9.  model.response        (orchestrator)
10. job.response_received (orchestrator)
11. wal.permission_check  (route.cloud.claude — governing, deliver_result → allowed, delivery_hold: true)
12. human.reviewed        (human) → accepted (releases delivery hold)
13. job.delivered          (phone_app)
```

**Sequence notes:**
- Steps 5–6: `select_model` has a `pre_action` gate at WAL-0, requiring human approval before routing proceeds. For single-route capabilities where classification deterministically selects the model, this check may be automatically satisfied by classification evidence — but the `wal.permission_check` event still fires.
- Steps 7 and 11: `dispatch_cloud` and `deliver_result` both have `pre_delivery` gates at WAL-0, encoded as `result: allowed, delivery_hold: true`. The action proceeds (cloud call executes, result is staged) but user-visible delivery is blocked until human review.
- Step 12: `human.reviewed` with `decision: accepted` releases the delivery hold. This is the same `human.reviewed` event used for post-delivery review at higher WAL levels — the `review_gate_context` field (see Section 3.5) distinguishes pre-delivery hold release from post-delivery feedback.

### 4.2 Override: Redirect (Post-Classification, Pre-Delivery)

```
ORIGINAL (J1, route.cloud.claude):
  1-8. [submitted → response_received]
  9. human.override       (redirect, detail: route.cloud.openai)
  10. job.failed           (redirected, failing_capability_id: route.cloud.claude)
  11. wal.demoted          (route.cloud.claude, trigger: override, demote 1)

SUCCESSOR (J2, parent: J1, route.cloud.openai):
  12. job.classified       (orchestrator, new governing_capability_id)
  13-18. [packaged → dispatched → response → delivered → reviewed]
```

### 4.3 Override: Post-Delivery Escalate (WAL-2)

```
ORIGINAL (J1, WAL-2):
  1-6. [submitted → dispatched → response → delivered (post_action)]
  7. human.override       (escalate, detail: route.multi)
  8. job.revoked           (escalation_supersede, successor: J2, superseded: [M1])

SUCCESSOR (J2, parent: J1, route.multi):
  9-14. [classified → packaged → dispatched ×2 → responses → delivered]
  15. human.reviewed       (accepted)
```

### 4.4 Context Packager Strip Failure (Egress Catch)

```
1-4. [submitted → classified → permission_check → packaged]
     (Packager MISSED a credential in the payload)
5.   context.strip_detail  (detected_by: egress_gateway, failing_capability_id: "context.package")
6.   egress.validation_failure (if strip blocked dispatch)
7.   job.failed            (context_error)
8.   wal.demoted           (context.package → SUSPENDED, trigger: strip_failure)
```

## 6. Querying

```bash
# All events for a job
jq 'select(.job_id == "019508a3...")' drnt-audit-2026-03-26.jsonl

# Strip failures caught by egress
jq 'select(.event_type == "context.strip_detail"
    and .payload.failing_capability_id != null)' drnt-audit-*.jsonl

# Permission checks with delivery holds
jq 'select(.event_type == "wal.permission_check"
    and .payload.delivery_hold == true)' drnt-audit-*.jsonl

# Held permission checks
jq 'select(.event_type == "wal.permission_check"
    and .payload.result == "held")' drnt-audit-*.jsonl

jq 'select(.event_type == "wal.demoted")' drnt-audit-*.jsonl

# Successor chain
jq 'select(.parent_job_id == "019508a3...")' drnt-audit-*.jsonl

jq 'select(.job_id == "..." and .event_type == "job.classified")
    | .payload.declared_pipeline' drnt-audit-*.jsonl

tail -f drnt-audit-2026-03-26.jsonl | jq '.'
```

## 7. Companion Stores

## 8. Write Path

Components provide `source_event_id` (self-generated UUIDv7), `timestamp` (at occurrence), and `durability` (per event type).

```
Component → Unix socket → Log Writer → dedup → assign event_id + committed_at + prev_hash → JSONL → fsync (if durable) → ACK
```

Watch/phone events transmit over Tailscale. If offline, events queue on-device and flush on reconnection. `source_event_id` and `timestamp` are set on-device; `committed_at` is set by the log writer. The gap between `timestamp` and `committed_at` is the offline queue duration.
